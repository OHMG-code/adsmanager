# repo-audit

## Purpose
Przeprowadź pełny audyt repozytorium przed rozpoczęciem zmian.

## Instructions
1. Rozpoznaj stack technologiczny projektu.
2. Ustal punkt wejścia aplikacji.
3. Znajdź błędy blokujące uruchomienie.
4. Wykryj TODO, FIXME, martwe ścieżki, brakujące integracje i niespójności.
5. Ustal, jak uruchamia się aplikację, testy, lint, build i migracje.
6. Przygotuj krótki plan dalszych działań z priorytetami.

## Output
Zwróć:
- wykryty stack,
- główne moduły projektu,
- błędy krytyczne,
- brakujące elementy,
- proponowaną kolejność napraw.
