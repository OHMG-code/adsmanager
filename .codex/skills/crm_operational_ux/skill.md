# Skill: CRM Operational UX

## Cel
Tworzenie systemu CRM jako narzędzia pracy, a nie tylko bazy danych.

## Zasady

1. CRM ma odpowiadać na pytanie:
   "Co użytkownik ma zrobić teraz?"

2. UI ma wspierać:
   - decyzje
   - rozmowy
   - działania

3. Nie projektuj jak SaaS.
   Projektuj jak narzędzie pracy zespołu.

## Priorytety

1. Czytelność
2. Szybkość działania
3. Dostępność funkcji
4. Kontekst

## Nie rób

- ukrywania funkcji dla minimalizmu
- dodawania funkcji bez kontekstu użycia
- przeładowania UI

## Wymagany proces

Przy każdej zmianie UI:
- co użytkownik widzi?
- co użytkownik zrobi dalej?
- czy wie co zrobić?

## Red flags

- „ładne, ale nie wiadomo gdzie kliknąć”
- brak następnego kroku
- zbyt dużo opcji bez hierarchii

## Output

- opis przepływu użytkownika
- uzasadnienie zmian UX
- wskazanie poprawy lub pogorszenia użyteczności
